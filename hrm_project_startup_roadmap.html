📌 Tips for success
  • อย่าข้ามขั้นตอนที่ 1-2 แม้ดูนาน ฐานรากชัดเจนช่วยส่วนข้างหน้าได้มาก
  • ในแต่ละ phase ให้สร้าง unit test ก่อนหรือพร้อมกับ feature (TDD optional)
  • ถ้า team มี 3+ คน แบ่ง parallel: ฝั่ง API (ขั้น 2-5), ฝั่ง UI (ขั้น 6-8)
  • อย่า perfectionist ขั้นแรก — release MVP แล้วค่อยปรับปรุง
